## File format & size

- CSV, comma-delimited, UTF-8, one carrier per row
- **~4.4M total entity rows** (all-time, includes inactive)
- Compressed: ~250-350 MB; Uncompressed: ~1.5-2 GB
- After active+for-hire+freight filter: **~180-230K rows** (matches operator expectation)
- Refresh cadence: daily for `az4n-8mr2`, monthly for legacy SMS files

## Filter recipe (active for-hire freight carriers)

**Server-side SoQL (preferred — avoids downloading 4M rows):**

```
$where=status_code='A' AND carrier_operation in('A','B') AND classdef in('Authorized For Hire','Exempt For Hire') AND power_units>=1 AND bus_units=0
```

**Client-side equivalent (if streaming full CSV):**

```python
keep = (
    row["STATUS_CODE"] == "A"                      # active
    and row["CARRIER_OPERATION"] in ("A", "B")     # interstate or intrastate (skip C if interstate-only)
    and row["CLASSDEF"] in (
        "Authorized For Hire",
        "Exempt For Hire",
    )                                              # excludes Private, Govt, Migrant, US Mail
    and (int(row["BUS_UNITS"] or 0) == 0)          # exclude pure passenger carriers
    and (int(row["POWER_UNITS"] or 0) >= 1)        # has at least 1 truck
)
```

## Recommended import approach

- **Stream-parse, don't full-load.** Socrata SoQL pagination: `$limit=50000&$offset=N` until empty. OR stream full CSV with `requests.get(stream=True)` + `csv.DictReader` on response iterator. 2 GB uncompressed will not fit comfortably in RAM on a small box.
- **Bulk-write to Mongo:** `pymongo` `bulk_write` with `UpdateOne(..., upsert=True)` keyed on `dot_number`, batches of 1000. Expect 5-15 min for full ~200K import.
- **Idempotent re-runs:** unique index on `dot_number`; let upsert handle daily delta. Add `last_imported_at` field for staleness detection.
- **Snapshot pinning:** save raw CSV (gz'd) to `D:\CHL_backups\fmcsa_census\YYYYMMDD.csv.gz` before each import — gives rollback + audit trail and respects the snapshot-after-save rule.

## Rate limits & access

- No auth, no API key required for `data.transportation.gov`
- Socrata applies a soft throttle (~1000 req/hr unauthenticated)
- Register a free Socrata app token (`X-App-Token` header) to lift the throttle — recommended for repeated paginated pulls
- No IP rate limit on bulk CSV endpoint, but a single full pull is ~250 MB — don't loop it
- Public-domain data; no license restrictions on redistribution

## Known gotchas

1. **MOTUS migration (2026):** column names may rename mid-year. Wrap column accessors in single mapping dict so rename is one-line fix.
2. **`CLASSDEF` is free-text, not a code.** Watch for "Authorized For Hire " trailing whitespace, capitalization drift. Normalize before matching.
3. **MC numbers are messy.** Carrier can have up to 3 dockets (DOCKET1/2/3) with prefixes MC, FF, MX. Most CHL workflows want MC docket — filter `DOCKETnPREFIX == 'MC'`.
4. **Some active carriers have `POWER_UNITS=0`** (paperwork-only / brokers misfiled as carriers). The `>=1` filter drops noise.
5. **Census file does NOT include operating-authority granted/revoked timestamps** — those are in L&I. "Active status" here means MCS-150 is current, not that authority is granted. **For broker-vetting you MUST cross-reference L&I (or QCMobile API) before dispatching loads.**
6. **Daily freshness ≠ real-time.** New authorities take 21 days to appear; revocations same. Don't rely on this for go/no-go on a load — use it for static directory + nightly sync.
7. **No email field on `az4n-8mr2`** (only on `kjg3-diqy`). If CHL wants email, do left-join on DOT_NUMBER from SMS subset.
