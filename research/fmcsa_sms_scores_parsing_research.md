# FMCSA SMS BASIC Scores — Parsing & Bulk-Ingest Research

> **Generated by:** sub-agent (general-purpose) dispatched by @dev-engineer 2026-05-07 in support of iter 142.1 nationwide-vetted-carrier-DB Phase 2 (SMS scores ingestion).
> **Scope:** how to programmatically retrieve SMS BASIC safety scores for any US motor carrier by USDOT number.

---

## URL pattern (verified via live fetches)

```
Overview:    https://ai.fmcsa.dot.gov/SMS/Carrier/{USDOT}/Overview.aspx
Per-BASIC:   https://ai.fmcsa.dot.gov/SMS/Carrier/{USDOT}/BASIC/{slug}.aspx
```

**USDOT (not MC) is the path key.** Tested DOTs 282191, 3331177, 4230738 → all returned valid SMS profiles. Bad DOT (e.g. `4569843` — operator's pre-FMCSA-authority number) silently redirects to SMS homepage rather than 404.

**Detection of bad DOT:** check response URL for `/Carrier/{DOT}/` OR check page body for `"Carrier Registration Details"` string. If absent → invalid DOT.

The 7 BASIC slugs (sub-pages from Overview):
- `UnsafeDriving`
- `CrashIndicator`
- `HOSCompliance`
- `VehicleMaint`
- `ControlledSubstances`
- `HMCompliance` (Hazmat)
- `DriverFitness`

## HTML structure (live samples)

**Overview.aspx** — uses unordered list of links to per-BASIC pages:
```html
<a href="/SMS/Carrier/282191/BASIC/UnsafeDriving.aspx">
  <img src="/SMS/images/Icons/BASICs/UnsafeDriving.png"> Unsafe Driving
</a>
<a href="/SMS/Carrier/282191/BASIC/CrashIndicator.aspx">
  <img src="..."> Not Public Crash Indicator
</a>
```

The `Not Public ` prefix is the **public-suppression flag** (Crash Indicator + HM Compliance are non-public for most carriers per FMCSA policy).

**Per-BASIC page** (e.g., `/BASIC/UnsafeDriving.aspx`) — DIV layout, no tables, no stable IDs/classes:
- Section heading: `<h3>On-Road Performance</h3>`
- Measure: `<strong>Measure: 0.00</strong>`
- Percentile: literal text `Percentile: 0%` (omitted ENTIRELY if insufficient data — no "Insufficient Data" string; absence is the signal)
- Alert icon: `<img src="/SMS/Images/Icons/prioritize_small.png" alt="Carrier may be prioritized for an intervention action and roadside inspection.">`
- Intervention threshold: text `50% Intervention Threshold` (varies: 50/60/65/80 per BASIC + carrier type)

**Lookback window** (Overview.aspx, verbatim):
```
Behavior Analysis & Safety Improvement Categories (BASICs) Based on a 24-month record ending April 24, 2026
```

**Inspections summary** (text run, not tabled):
```
# Insp. OOS Driver Inspections 8 0.0% Vehicle Inspections 58 0.0%
```

**Crashes summary**:
```
Fatal Crashes 0 Injury Crashes 0 Towaway Crashes 0
```

## Recommended extraction patterns

```python
RE_LOOKBACK   = r"Based on a 24-month record ending\s+([A-Z][a-z]+ \d{1,2}, \d{4})"
RE_MEASURE    = r"Measure:\s*</?[^>]*>?\s*([\d.]+)"
RE_PERCENTILE = r"Percentile:\s*</?[^>]*>?\s*(\d{1,3})%"
RE_THRESHOLD  = r"(\d{1,3})%\s*Intervention Threshold"
RE_ALERT      = r"prioritize_small\.png"          # presence = alert
RE_NOT_PUBLIC = r"Not Public\s+(Crash Indicator|Hazardous Materials Compliance|HM Compliance)"
RE_INSP       = r"Driver Inspections\s+(\d+)\s+([\d.]+)%\s+Vehicle Inspections\s+(\d+)\s+([\d.]+)%"
RE_CRASH      = r"Fatal Crashes\s+(\d+)\s+Injury Crashes\s+(\d+)\s+Towaway Crashes\s+(\d+)"
RE_BASIC_LINK = r"/SMS/Carrier/(\d+)/BASIC/(\w+)\.aspx"
```

**Recommended approach:** Use BeautifulSoup with `lxml` parser to strip tags first, then regex on text. Pure regex on raw HTML works but is more fragile due to variable whitespace inside `<strong>`/`<span>` wrappers.

**BeautifulSoup is NOT currently a backend dependency.** Decision: install it for this work (mature, MIT license, ~1MB) OR keep regex-only with extra defensive whitespace patterns.

## Edge cases (observed)

- **Insufficient data:** percentile line is SIMPLY ABSENT — no marker. Detect by `RE_PERCENTILE` not matching while measure is present.
- **Not Public BASICs:** Crash Indicator + HM Compliance are non-public for most carriers; carry the `Not Public` text prefix on the Overview link. Honor this — don't rely on these scores being available.
- **New / zero-inspection carriers:** measure = `0.00`, no percentile, no alert. Distinguish "clean" from "no data" by inspection count (>0 = clean, 0 = no data).
- **Bad DOT:** site redirects to homepage rather than 404 — verify response by presence of `Carrier/{DOT}/` in canonical URL or DOT echo in page.
- **Passenger vs property carriers** have different intervention thresholds (50% passenger, 65/80% property, 60% HM) — extract `RE_THRESHOLD` per page rather than hardcoding.

---

## Fragility points

- DIV-based layout with NO stable IDs/classes — relying on literal text labels (`"Measure:"`, `"Percentile:"`, `"Intervention Threshold"`, `"24-month record ending"`) is the practical anchor. These have been stable for 5+ years per third-party scraper docs but **are not contractual**.
- Path casing: `/Images/` vs `/images/` inconsistency → use case-insensitive matching.
- Inspection/crash summary is **untabled text** — whitespace is the only delimiter; use `\s+` not literal spaces.
- Monthly refresh (3rd Friday of month) — cache TTL ≥ 7 days is safe; daily polling wastes calls.

## ⭐ STRONG ALTERNATIVE: Bulk Monthly CSV Download

**FMCSA publishes the same SMS input data as monthly CSV downloads at:**
```
https://ai.fmcsa.dot.gov/SMS/Tools/Downloads.aspx
```

**For bulk/recurring carrier scoring, ingest the monthly file ONCE and avoid scraping entirely.** Reserve the per-carrier page-scrape path for ad-hoc lookups of carriers not yet in the snapshot.

**Implication for iter 142.1 Phase 2 architecture:**
- **Monthly bulk:** download SMS scores CSV, bulk-update `db.carriers.sms.*` for all carriers in one pass. Rate-limit-free. Refreshes monthly automatically.
- **Per-carrier on-demand:** scraping endpoint reserved for new carriers added between monthly snapshots, OR for live/fresh checks.

Recommend defaulting to bulk monthly + on-demand fallback. This eliminates the 75-day refresh-time concern from the per-carrier model.
