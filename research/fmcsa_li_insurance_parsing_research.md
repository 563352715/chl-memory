## Tier 2 (FALLBACK): L&I portal HTML scraping

**The L&I portal uses Oracle PL/SQL Web Toolkit** (the `PKG_*.prc_*` URL pattern). It is **NOT a simple URL-by-DOT lookup**. Multi-step required:

| Step | Endpoint | Method | Key params |
|------|----------|--------|------------|
| 1 | `https://li-public.fmcsa.dot.gov/LIVIEW/pkg_carrquery.prc_carrlist` | GET | (load form, get session cookie + challenge) |
| 2 | same URL | **POST** | `n_dotno=<USDOT>` OR `pv_pref_docket=MC<num>`, `pv_vpath=LIVIEW`, `pv_choosen_question`, `pv_xanswer` (CAPTCHA answer) |
| 3 | `pkg_carrquery.prc_getdetail` | GET | `pv_apcant_id=<internal_id>&pv_vpath=LIVIEW` |
| 4 | `pkg_carrquery.prc_activeinsurance` | GET | `pv_apcant_id=<internal_id>&pv_pref_docket=MC<num>&pv_vpath=LIVIEW` |
| 5 | `pkg_carrquery.prc_insfiler_details` | GET | `pv_inser_id=<id>&pv_apcant_id=<id>&pv_prc_back=prc_activeinsurance` |
| 6 | `pkg_carrquery.prc_inshistory` | GET | `pv_apcant_id=<internal_id>` |

**Critical:** `pv_apcant_id` is L&I internal primary key — NOT the USDOT. Must scrape from step 2 results page.

## CAPTCHA / Anti-scraping (CONFIRMED)

The L&I search form requires a "challenge question" — verified verbatim on `prc_carrlist`. Sub-agent attempts to GET deep links directly all bounced back to search form, confirming portal enforces:
- Session cookie set on form load
- Challenge-question answer required to submit search
- Likely referrer check on deep links

Challenge questions appear to be a small rotating set (e.g., "What is 2+3?"). Community scrapers handle via hardcoded answer table or by parsing `pv_choosen_question` and matching against dictionary. **No reCAPTCHA**, no JS execution required — plain `requests.Session()` works once challenge handled.

## Active Insurance HTML structure (when Tier 2 fallback needed)

Active Insurance page (`prc_activeinsurance`) renders a `<table>` with these columns:
```
Form | Type | Insurance Carrier | Policy/Surety | Posted Date | Coverage From | Coverage To | Effective Date
```

- **Form column:** `BMC-91` / `BMC-91X` (BIPD), `BMC-34` (Cargo), `BMC-32` (Bond)
- **Type column:** text "BIPD/Primary", "Cargo", etc.
- **Insurance Carrier column:** hyperlink — `<a href="pkg_carrquery.prc_insfiler_details?pv_inser_id=NNNNN...">INSURER NAME</a>`
- **Coverage From/To:** dollar amounts like `$1,000,000` (BIPD floor $750K) and `$100,000` (cargo)

## Tier 2 extraction (BeautifulSoup recommended)

```python
soup = BeautifulSoup(html, "lxml")
# Active insurance table is the one whose <th> row contains "Form" + "Coverage From"
ins_table = next(t for t in soup.find_all("table")
                 if t.find("th", string=re.compile(r"Coverage From")))
for row in ins_table.find_all("tr")[1:]:
    cells = [td.get_text(strip=True) for td in row.find_all("td")]
    form, ins_type, insurer, policy, posted, cov_from, cov_to, eff = cells
    insurer_id = re.search(r"pv_inser_id=(\d+)", row.find("a")["href"]).group(1)
```

Don't regex the whole page — table position shifts when carrier has pending applications above active block.

## AM Best insurer rating lookup

- Official rating-feed API is **paid** (enterprise contract).

**Pragmatic path for "flag below A" rule:**

- Refresh quarterly by hand
- Cheaper and more reliable than scraping ambest.com

## Edge cases (observed)

- **Multiple BIPD filings:** `prc_activeinsurance` shows ALL active; most recent `Posted Date` is binding. Sort by that, take row 0.
- **Cancelled coverage:** appears only on `prc_inshistory`, not active page. Active page being empty = uninsured.
- **Below federal minimum:** still listed; just compare `cov_from` parsed to int against `750_000` (BIPD) and `100_000` (cargo) thresholds in code.
- **"Reject" filings:** separate table on detail page; ignore for vetting.

## Fragility points (Tier 2 only)

1. **CAPTCHA dictionary drift** — if FMCSA rotates challenge questions, scraper breaks silently. Add alarm if `prc_activeinsurance` returns form instead of table.
2. **`pv_apcant_id` not stable across redomicile** — always re-resolve from USDOT, never cache long-term.
3. **24-hour stale window** — FMCSA states new filings don't appear for 24h after submission. Don't dispatch on same-day-submitted carrier.
4. **Rate limits** — undocumented but observed ~1 req/sec ceiling in community scrapers. Add 1.5s sleep between carriers.
5. **Oracle outages** — portal goes down ~monthly for maintenance, returns generic "Service Temporarily Unavailable". Wrap with retry + fallback.
