## Proposed enhancements

### Layer 1 — Vetting Pipeline Tiles (HIGHEST PRIORITY)

Add a SECOND row of tiles BELOW the equipment breakdown showing carriers by vetting state. This is the main value-add the operator asked for ("Lists of carriers that are vetted or at different stages of being vetted"):

```
┌─────────────────────────────────────────────────────────────────┐
│  PROSPECT  SAFER_VETTED  CONTACTED  COI_PENDING  BCA_PENDING   │
│     12         8             3          2            1         │
│                                                                 │
│  VETTED_COMPLETE   DECLINED   NEEDS_REVET                      │
│        5              4           2                             │
└─────────────────────────────────────────────────────────────────┘
```

Each tile is clickable → filters the carrier list below to that vetting state. Color-coded:
- 🔘 PROSPECT — gray (just imported, no vetting started)
- 🔵 SAFER_VETTED — blue (FMCSA data pulled, no contact yet)
- 🟡 CONTACTED — yellow (initial call/email made)
- 🟠 COI_PENDING — orange (awaiting Certificate of Insurance)
- 🟠 BCA_PENDING — orange (awaiting Broker-Carrier Agreement signature)
- 🟢 VETTED_COMPLETE — green (ready for dispatch)
- ⚪ DECLINED — gray (vetting failed, do not re-vet)
- 🔴 NEEDS_REVET — red (data >30 days old OR insurance expired soon)

### Layer 2 — "Add Carrier" prominent button

Currently there's no obvious way to add a carrier. Add a primary CTA button top-right:

```
[ + Add Carrier ]   [ + Bulk Import (CSV) ]
```

**Add Carrier flow:**
1. Click → modal opens
2. Operator pastes MC number (or DOT number)
3. Modal shows "Looking up FMCSA SAFER..." spinner
4. ~5 seconds later, modal shows pre-filled carrier card:
   - Legal name
   - Operating status (color-coded)
   - Insurance amounts + filing date
   - SMS BASIC scores (with alert flags)
   - Recent crash count
   - Address + phone
5. Operator reviews, can edit notes field
6. Click "Add to Roster" → carrier added with `vetting_status: "SAFER_VETTED"`
7. Modal closes, list refreshes, vetting tiles update

**Bulk Import flow:**
- Operator pastes multiple MCs (one per line, or comma-separated)
- System runs SAFER lookups in parallel (~30 sec for 10 carriers)
- Shows table preview with all results + red-flag highlights
- Operator confirms which to add (checkboxes)
- Bulk add to roster

### Layer 4 — Carrier Detail Side Panel

Click any row → side panel slides in from the right (don't navigate away from the list). Shows:

**Tab 1: Overview**
- Full SAFER snapshot (legal name, MC, DOT, address, phone, equipment, fleet size)
- Operating Status (color-coded indicator)
- Insurance summary table (BIPD amount + filing date, Cargo amount + filing date, Insurer name + AM Best rating)
- SMS BASIC scores breakdown (7 categories with percentile bars)
- Crash + inspection summary

**Tab 2: Vetting Workflow**
- Current vetting state with timestamp
- Checklist of completed steps (matches `iter_142_1_carrier_vetting_checklist.md`):
  - [✓] Step 2: SAFER lookup (2026-05-07 14:23)
  - [✓] Step 3: SMS scores (2026-05-07 14:23)
  - [✓] Step 4: Crash history (2026-05-07 14:23)
  - [ ] Step 5: First contact
  - [ ] Step 6: COI received
  - [ ] Step 7: BCA signed
  - [ ] Step 8: W-9 received
- Action button: "Mark next step complete →"

**Tab 3: Activity Log**
- Reverse-chronological timeline of all events for this carrier:
  - 2026-05-08 09:15 — SMS sent: "Load opportunity ATL→DAL..."
  - 2026-05-07 16:40 — Note added: "Dispatcher confirmed availability for Tuesday pickup"
  - 2026-05-07 14:23 — Imported via FMCSA SAFER lookup
- Free-text note input at bottom

**Tab 4: Documents**
- COI upload (drag-drop or click to upload PDF)
- BCA upload (signed PDF)
- W-9 upload
- All docs preview-viewable inline

**Tab 5: Recent Loads** (post-iter-141.2)
- Loads dispatched to this carrier in last 90 days
- On-time delivery %
- Total revenue from this carrier
- Average margin

### Layer 5 — Insurance Expiry Alert Strip

At top of tab (above equipment tiles), if any carriers have insurance expiring in <30 days:

```
🔴 3 carriers have insurance expiring within 30 days.
   [ View list → ]
```

Click → filters list to show only those carriers.

### Layer 6 — Smart Filters (replace simple dropdowns)

Current filters: equipment, state, safety_rating. Expand to:

- **Equipment** (multi-select): Reefer + Dry Van combo, etc.
- **State** (multi-select)
- **Vetting State** (multi-select)
- **Lane Served** (text input — "MO→TX") — uses `lanes_served` field
- **Last Activity** (date range — "active in last 30 days")
- **Insurance Expiry** (date range — "expires before 2026-06-30")
- **Fleet Size** (range slider — 1-10, 11-50, 51+)
- **Years in Business** (range slider)
- **SMS Alert Categories** (multi-select — "no Crash Indicator alerts")
- **Has signed BCA** (toggle yes/no)

**Save filter presets:** "My MO/TX Reefer Carriers" preset that operator can pin and reuse.

### Layer 7 — Bulk Actions

Checkbox on each row + bulk action bar:

- Re-vet selected (re-fetch SAFER for all selected)
- Mark contacted (bulk transition)
- Export selected to CSV
- Send outreach SMS (when iter 142.1 stage 1d ships)

### Layer 8 — Acquisition Channel Tracking

Add field `acquisition_channel` to db.carriers:
- "load_board_response" (DAT)
- "load_board_response" (Truckstop)
- "cold_outreach" (operator-initiated via FMCSA Census filter)
- "referral"
- "manual_add"

Show in tile breakdown: "Acquisition this month: 8 from load board, 3 cold, 1 referral"

Helps operator see which channels yield best carriers (highest VETTED_COMPLETE %).

### Layer 9 — Census-Powered Carrier Discovery

(Goes BEYOND "what we have" — addresses the cold-search question from earlier conversation.)

Add a new sub-button alongside Pre-Vet Pipeline + Carrier Hub:

```
[ Pre-Vet Pipeline ]  [ Carrier Hub ]  [ + Find Carriers ]
```

**Find Carriers flow:**
1. Click → modal "Find New Carriers"
2. Operator selects: equipment type(s) + base states + fleet size range + max safety percentile
3. System queries FMCSA Census file (cached, refreshed monthly)
4. Returns 50-200 candidate carriers matching criteria
5. Auto-runs SAFER + SMS lookup on top 20 (~10 min wall-clock)
6. Shows ranked list with vetting summary preview
7. Operator selects which to add (checkboxes), bulk adds to roster as `acquisition_channel: cold_outreach`

This is the proactive lane-targeted carrier discovery operator asked about ("search for carriers... Without using load board").

### Layer 10 — Per-Carrier Health Monitoring

Background job (extends iter 141.1 stage 1c lane_scoring_cron pattern):
- Daily: re-fetch SAFER for any carrier whose data is >7 days old
- If status changes (insurance lapses, OOS order issued, authority revoked) → flip carrier to `NEEDS_REVET` + push notification to operator
- Monthly: full SAFER + SMS refresh for all `VETTED_COMPLETE` carriers

Show in UI:
- Top-strip alert: "⚠️ Status changed for 2 carriers since last check"
- Per-row indicator: small dot — green if data <7 days old, yellow 7-30 days, red >30 days
