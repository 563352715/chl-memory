## 1. Faults found ‚Äî REAL platform bugs

### BUG #2 confirmed: geofence not enforced (pickup + delivery)
- **Detected by:** SOP cycles 10, 31 (`fi_geofence_pickup`) + cycles 11, 32 (`fi_geofence_delivery`)
- **Symptom:** driver advanced through pickup/delivery from synthesized GPS coordinates >100yd outside the 50yd inclusive radius. Platform allowed `at_pickup` / `at_delivery` transitions without rejecting the GPS ping.
- **Expected:** `failure_code='geofence_violation'`, transition rejected with HTTP 412
- **Actual pickup:** `actual_failure=None` ‚Äî silently allowed
- **Actual delivery:** `actual_failure='invalid_transition'` ‚Äî wrong reason code (delivery hit a different guard)
- **Fix needed:** wire 50yd inclusive radius check at `/api/public/carrier-offer/{token}/advance-status` before applying the state-machine transition, mirroring the LLL14 POD-gate pattern. Use `captured_lat/lng` from the advance-status payload vs `origin_lat/lng` (pickup) or `destination_lat/lng` (delivery).
- **Severity:** **auto-fixable** ‚Äî clear pattern, design locked, low risk. Will ship next batch.

### BUG #4 confirmed: cancel-load returns 500
- **Detected by:** SOP cycles 18, 39 (`fi_cancel_load_tonu`)
- **Symptom:** TONU cancel flow fires `actual_failure='server_error'` on cancel attempt instead of clean `cancelled` transition
- **Severity:** auto-fixable pending root-cause investigation
- **Status:** queued for next iter ‚Äî needs server-side log inspection at the cancel endpoint

### BUG #6 NEW: bad-BOL image-quality validator not firing
- **Detected by:** SOP cycles 15, 36 (`fi_bad_bol_upload`)
- **Symptom:** Driver uploads a deliberately corrupted BOL image; expected `image_quality_rejected` (412); got `pod_required` (subsequent gate). Means the BOL was accepted and only the downstream POD gate fired.
- **Root cause hypothesis:** `image_quality` validator either disabled for synthetic loads OR not gated on BOL upload (only POD upload). Need to inspect `/api/public/carrier-offer/{token}/bol-upload` for image_quality call.
- **Severity:** auto-fixable pending investigation
- **Status:** queued for next iter

### BUG #7 NEW: wrong-consignee-PIN returns 409 'conflict' instead of 'bad_pin'
- **Detected by:** SOP cycles 17, 38 (`fi_wrong_consignee_pin`)
- **Symptom:** Driver submits wrong consignee PIN; platform correctly blocks but returns HTTP 409 with code `conflict` instead of dedicated `bad_pin` code.
- **Severity:** **cosmetic** ‚Äî security-correct (rejection happens), but error-code inconsistency hurts driver-UI messaging
- **Auto-fix:** add a dedicated `BadConsigneePinError(409, 'bad_pin')` subclass to standardize the error contract
- **Status:** queued ‚Äî low-priority cosmetic

### BUG #8 NEW: synthesizer not applying broker margin (market_rate == carrier_rate)
- **Detected by:** Net profit calculation showed `margin_spread_delivered = $0.00`. All 75 loads have `market_rate == rate`.
- **Implication:** test data shows zero broker margin, which makes net-profit reporting from this run unrealistic
- **Root cause:** `synthetic_pipeline_runner.py::_build_load_spec` LLL15 update writes `rate` but doesn't compute and write a market_rate above it
- **Severity:** **synthesis bug, not platform** ‚Äî production rate-calc engine DOES apply `market_adjustment = 1.05` when called via API. Synthesizer is bypassing the API and storing computed `rate` directly into both fields.
- **Auto-fix:** in `_build_load_spec`, after computing `rate`, set `market_rate = round(rate * 1.05, 2)` (matches the engine's broker-spread)
- **Status:** queued ‚Äî affects all profit reporting; should fix before next run
